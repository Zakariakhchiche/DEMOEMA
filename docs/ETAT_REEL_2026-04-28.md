## 4. Architecture codegen — état au 28/04

### Bronze codegen (`ingestion/codegen.py` + `bronze_bootstrap.py`)

```
spec YAML → format auto-detect cascade → choix template/référence → LLM lead-data-engineer
                                                                    → 2 itérations max
                                                                    → endpoint test (HEAD/GET probe)
                                                                    → validation Python AST
                                                                    → write sources/{sid}.py
                                                                    → register in SOURCES dict
```

**9 formats supportés** (vs 4 avant 28/04) :
- `rest_json` (template `rest_json_ods.py.tmpl`)
- `csv` / `csv_gz` (template `csv_dump.py.tmpl`)
- `zip` (template `zip_csv.py.tmpl`, ref `_zip_csv_ref.py`)
- **`tar_gz_xml`** ⭐ (ref `_tar_gz_xml_ref.py`) — DILA OpenData
- **`xml_single`** ⭐ (ref `_xml_single_ref.py`) — gros XML monobloc avec `lxml.iterparse`
- **`parquet`** ⭐ (ref `_parquet_ref.py`) — `pyarrow.parquet.iter_batches`
- **`jsonl`** ⭐ (ref `_jsonl_ref.py`) — JSONL streaming + auto-decompress `.gz`
- `geojson` (LLM adapte avec hint)

**Parallélisme** : `bronze_codegen_parallelism = 4` (config). À chaque tick (5 min) : asyncio.gather sur tous les missing avec Semaphore(4) — 1 spec / tick → 4 specs / tick.

### Silver codegen (`ingestion/silver_codegen.py` + `silver_engine.py`)

**Topologique parallèle** (commit 2932c77) :

```
_topo_levels_specs() → list[list[str]] (Kahn par niveaux)
  level 0 : silvers à sources bronze pures (parallel build OK)
  level 1+ : silvers-of-silver (attendent leurs deps)

bootstrap_missing_silvers() :
  pour chaque level :
    asyncio.gather(*[_build_one_silver(s, sem) for s in level])
    Semaphore(silver_codegen_parallelism=4)
    rafraîchit `existing` après chaque level
```

**Mesuré** : 14 silvers à 5 min/spec en série = 70 min → **~20 min en parallèle 4-wide**.
