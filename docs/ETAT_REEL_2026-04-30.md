## 3. Scoring v3 PRO — 4 axes business

Refonte complète du scoring (avant : barème additif 13 dim / 123 signaux qui
produisait 50 % des entreprises ≥ 65 — statistiquement absurde). Nouveau modèle :

```
deal_score = (transmission × attractivity × scale)^(1/3) × risk_multiplier
```

| Axe | 0-100 | Logique |
|---|---:|---|
| **TRANSMISSION** | 0-100 | proba cession (âge dirigeant 55-75 sigmoïde, n_sci ≥ 2 + capital, late_filing, homme-clé) |
| **ATTRACTIVITY** | 0-100 | valeur réelle (proxy_margin × multiple sectoriel Argos 2024) |
| **SCALE** | 0-100 | barrière transactionnelle (CA ≥ 5M small cap → ≥ 100M large) |
| **STRUCTURE** | 0-100 | suitability (SAS/SA, has_pro_ma, multi-mandats, holding) |
| **RISK** | × 0..1 | multiplicateur (sanction OFAC/UE = ×0, CNIL ×0.75, contentieux ×0.85) |

## 5. Codegen tools — silver/gold autonomes

Module `infrastructure/agents/platform/ingestion/codegen_tools.py`.
8 tools READ-ONLY exposés au LLM lors de la génération du SQL silver/gold :

| Tool | Usage |
|---|---|
| `introspect_table(schema, table OR tables[])` | colonnes + types réels (mode multi pour 1 tool call sur N tables) |
| `find_column(column_name)` | recherche dans quelles tables existe une colonne |
| `sample_jsonb_keys(schema, table, column)` | top-keys d'un payload jsonb |
| `peek_sample_rows(schema, table, n)` | 3-5 vraies lignes en JSON |
| `count_rows_estimate(schema, table)` | rows estimés via pg_class |
| `look_at_existing_silver(silver_name)` | SQL d'une autre silver appliquée (template) |
| `dry_run_sql(select_sql)` | EXPLAIN avant apply |
| `test_compile_create(create_sql)` | TEST FINAL — CREATE TEMP VIEW + DROP |

Cadrage strict :
- 8 tools READ-ONLY, regex anti-injection (`schema/table/column`)
- `statement_timeout = 5-12s` par tool call
- `max_iterations = 8` par silver (cap LLM)
- `tool_audit` persisté dans `audit.silver_specs_versions.tool_calls jsonb`
- Toggle env `CODEGEN_USE_TOOLS=1` (default ON)

Activé pour silver_codegen + gold_codegen via `bootstrap_missing_*`.
