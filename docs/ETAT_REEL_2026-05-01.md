Tests : `test_no_label_hallucination.py` (paramétré sur 6 cas DL28/46/57/87/92/94) + `test_bodacc_filters.py` (10 tests filtres).

## 3. Bugs reportés (suivis non patchés cette session)

### SCRUM-122 (P1) — Sync dashboard ↔ fiche gold-only [follow-up G4 part 2]

EQUANS dashboard 285.7 M€ vs fiche 152.4 M€ (-46 %). Cause : `_cibles_from_silver` cap le score à 95 et lit `silver.inpi_comptes` (millésime différent) alors que `_cibles_from_gold` lit `gold.entreprises_master`. Patch nécessite télémétrie pré-merge :

```sql
SELECT COUNT(*) FROM silver.inpi_comptes c
WHERE NOT EXISTS (SELECT 1 FROM gold.entreprises_master m WHERE m.siren = c.siren);
-- alerter si > 1 % gap
```

Risque : SIREN actif silver mais absent gold = fiche vide (régression user-visible). PR séparée prévue après audit télémétrie.

### G2 (P0) — Fiche Pappers → silver datalake [bloqué]

Mon plan G2 entre en collision avec la branche `origin/chore/remove-pappers` (commit `bae73c0` non mergé). Cette branche retire le token Pappers et le dead flag `PAPPERS_MCP_URL`, mais ne migre pas la logique fiche du SIREN lookup vers le datalake silver. Mon G2 dépend de cette branche pour ne pas dupliquer.

**Action recommandée** : merger d'abord `chore/remove-pappers`, puis ouvrir G2 (PR séparée) qui remplace `get_pappers_company` lookup par `httpx.get('/api/datalake/fiche/{siren}')` interne dans `main.py:1604-1656`.

### G5 (P0) — SSE race condition frontend [estimation 3-4 j]

Refacto majeur du consumer SSE de `ChatPanel.tsx:374-406` : extraire dans `useSseStream(query, signal)` hook avec `useReducer`. Remplacer `Promise.all` par `await sequentially` + `flushSync`. Ajouter `streamId` ref pour invalider les setState d'un stream obsolète. Tests Playwright `chat-no-loss.spec.ts`.

Bug confirmé en audit : 5 questions consécutives → API SSE backend complète (200), DOM ne rend pas. Preuve : `dl1_response.network-response` contient le payload PERMIS INFORMATIQUE complet, le DOM affiche EMERIA (réponse précédente).

PR séparée à prévoir, plus risqué que les autres G car React 19 + flushSync + tests Playwright stables = 3-4 j en pratique.
